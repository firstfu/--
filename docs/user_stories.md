# 花錢找人一起爬山系統 - 使用者故事與流程

## 1. 使用者故事

### 作為付費方（尋找爬山夥伴者）

1. **新手登山者小明**

   > 身為一個登山新手，我想找一位有經驗的登山夥伴帶我爬玉山，這樣我就能安全地完成人生第一次高山冒險，並學習正確的登山技巧。

   **驗收標準**：

   - 能夠發佈詳細的爬山需求，包括路線、日期和預算
   - 可以檢視潛在夥伴的經驗和專長
   - 能夠與有經驗的登山者溝通細節

2. **獨行俠陳小姐**

   > 我經常獨自一人爬山，但有些路線較為危險，我希望能找到可靠的夥伴一同前往，提高安全性，同時也能分享旅途的樂趣。

   **驗收標準**：

   - 能夠快速搜尋有空的登山夥伴
   - 可以查看對方過去的登山經驗和評價
   - 能夠提前協調登山計劃

3. **攝影愛好者阿強**

   > 身為一名風景攝影師，我需要找人陪我上山拍攝日出，最好是熟悉路線且願意在天還沒亮時出發的夥伴。

   **驗收標準**：

   - 能夠在需求中詳細描述特殊要求（如攝影、清晨出發等）
   - 可以根據特定技能或特長篩選夥伴
   - 能夠分享期望的攝影地點和時間

4. **資深登山家老王**

   > 雖然我有豐富的登山經驗，但我希望找個有趣的年輕人陪伴我一起爬山，這樣旅途中能有人聊天，互相拍照，分享樂趣。

   **驗收標準**：

   - 能夠指定偏好的夥伴類型（如年齡、興趣）
   - 可以瀏覽夥伴的個人資料和照片
   - 能夠選擇具有良好溝通能力的夥伴

5. **都市白領麗莎**

   > 我想找個外表吸引人、健談的登山夥伴，陪我週末去郊外爬些簡單的山。我有些登山經驗，但更注重旅途中的社交互動和拍攝美麗的照片。

   **驗收標準**：

   - 能夠查看潛在夥伴的照片和自我介紹
   - 可以根據興趣和個性篩選夥伴
   - 能夠透過平台先行溝通了解對方

### 作為受雇方（提供陪伴服務者）

1. **專業登山嚮導小華**

   > 我擁有多年登山經驗和專業證照，希望利用閒暇時間帶領新手登山，一方面分享知識，另一方面賺取額外收入。

   **驗收標準**：

   - 能夠展示個人資料、經驗和專業證照
   - 可以瀏覽並篩選符合自己專長的爬山需求
   - 能夠與潛在客戶溝通行程細節

2. **休閒登山愛好者小林**

   **驗收標準**：

   - 能夠設定自己可提供服務的時間和路線
   - 可以靈活選擇接受的爬山邀約
   - 能夠簡單地與對方協調行程

   - 能夠發佈團體活動並設定收費標準

   **驗收標準**：

   - 能夠展示個人健身專業知識和相關照片

   - 能夠被對專業健身指導有需求的人找到

5. **大學生小美**

   > 我是個活潑開朗的大學生，喜歡戶外活動和拍照。我希望透過提供爬山陪伴服務，既能賺取部分學費，又能認識不同的人和拓展人脈。

   **驗收標準**：

   - 能夠展示個人特質和照片
   - 可以設定自己提供服務的特色（如擅長拍照、健談等）
   - 能夠根據個人時間靈活接單

## 2. 核心使用流程

### 付費尋找爬山夥伴流程

1. **註冊/登入平台**

   - 填寫基本資料
   - 設定個人偏好（登山經驗、體能水平等）
   - 上傳個人照片和自我介紹

2. **發佈爬山需求**

   - 選擇目標路線/山區
   - 設定日期和時間
   - 描述特殊需求（如帶領新手、攝影等）
   - 設定預算範圍
   - 指定需要的人數
   - 指定偏好的夥伴類型（可選）

3. **瀏覽並選擇潛在夥伴**

   - 查看符合條件的夥伴列表
   - 檢視夥伴詳細資料（照片、經驗、評價等）
   - 發送邀請或直接聯繫
   - 透過平台進行初步溝通

4. **確認行程細節**

   - 與夥伴討論具體行程
   - 協商價格和服務內容
   - 確認最終方案
   - 交換必要的聯繫方式

5. **完成行程及評價**
   - 完成爬山活動
   - 給予夥伴評價和反饋
   - 分享體驗（可選）
   - 建立長期關係（可選）

### 提供爬山陪伴服務流程

1. **註冊成為服務提供者**

   - 填寫詳細的登山經驗
   - 上傳相關證照（若有）
   - 上傳個人照片和特色描述
   - 設定服務範圍和價格區間

2. **設定可提供服務的時間**

   - 標記可接單的日期
   - 設定偏好的路線或地區
   - 描述自己的特色和優勢

3. **瀏覽需求並申請**

   - 查看符合自己時間和專長的需求
   - 向需求方提出服務申請
   - 簡要說明自己的優勢和報價
   - 與潛在客戶進行初步溝通

4. **溝通確認細節**

   - 與需求方討論具體安排
   - 確認安全措施和應急計劃
   - 達成最終協議
   - 了解客戶特殊需求（如拍照需求、社交期望等）

5. **提供服務並收取回饋**
   - 完成爬山陪伴服務
   - 收集客戶評價
   - 累積經驗和信譽
   - 建立回頭客關係

## 3. 交互場景示例

### 場景一：新手尋找指導

> 小明是一位登山新手，想要嘗試爬玉山但缺乏經驗。他在平台上發佈了需求，希望找一位有經驗的嚮導。小華看到這個需求後，提出了服務建議，包括行前訓練和裝備指導。雙方協商後確定了價格和行程，成功完成了一次安全且愉快的登山體驗。

### 場景二：臨時尋找夥伴

> 陳小姐計劃週末爬合歡山，但原本約好的朋友臨時無法前往。她在平台上發佈了緊急需求，希望找到已經有爬山計劃的小團體加入。小林正好也打算同一天去合歡山，於是接受了邀請，以較低的價格提供了陪伴服務，兩人成為了登山好友。

### 場景三：特殊需求配對

> 阿強是專業攝影師，需要在特定時間點到玉山主峰拍攝日出。他在平台上詳細說明了需求，包括凌晨出發、攜帶攝影器材等特殊要求。專業嚮導小華接受了這個挑戰，憑藉熟悉路線的優勢，成功帶領阿強在最佳時間點到達拍攝地點。

### 場景四：社交需求滿足

> 老王是一位退休的資深登山家，雖然對路線非常熟悉，但希望找個年輕人陪伴自己爬山聊天。他在平台上發佈需求，強調希望找個健談、有趣的夥伴。大學生小美看到後申請提供服務，憑藉開朗的性格和良好的溝通能力，給老王的登山之旅帶來了很多樂趣，兩人還約定下次繼續合作。

### 場景五：形象與陪伴需求

> 麗莎是一位都市白領，熱愛在社交媒體分享生活點滴。她希望找個外表吸引人的夥伴一起爬山並拍攝精美照片。她在平台上發佈需求時特別注明了對夥伴外表和拍照能力的要求。健身教練阿傑接受了這個需求，不僅提供了專業的健行指導，還幫麗莎拍攝了多張精美的照片，使她的社交媒體獲得了大量讚好。

## 4. MVP 階段重點關注的流程

在 MVP 階段，我們將重點關注以下核心流程，以驗證產品概念：

1. **基本註冊和個人資料建立**（包括照片上傳和特質展示）
2. **發佈和瀏覽爬山需求**（含夥伴偏好設定）
3. **簡易的夥伴配對機制**
4. **基礎的用戶互動功能**
5. **個人特質與社交需求的表達與匹配**

其他進階功能如支付整合、詳細評價系統等將在後續版本中開發。
